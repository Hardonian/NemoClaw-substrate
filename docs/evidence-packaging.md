# Evidence Packaging

NemoClaw's governed substrate requires all operational evidence to be packaged deterministically. This guarantees that audit trails, execution histories, and receipts cannot be forged or mutated after the fact without detection.

## Append-Only Semantics

Evidence logs must adhere to strict append-only semantics:
- Once an entry is generated and hashed, it is immutable.
- New events are appended to the ledger via deterministic arrays.
- The state of the world is represented by the continuous sequence of entries.

## Stable Hashing

All evidence artifacts are hashed using a stable, key-sorted JSON representation (SHA-256). This ensures that structural representation details (like object key order) do not alter the integrity verification hash.

## NDJSON Export

For streaming scenarios or log aggregators, evidence is typically exported in NDJSON format. Each line represents a deterministic JSON string representing an `ExportEntry`.

## Types of Evidence

- `receipt`: Proof of action completion.
- `execution-plan`: The intended structural plan generated by the agent.
- `approval-lineage`: Human or operator approvals tracing back to authorization.
- `telemetry-summary`: Rollup of metrics and performance data.
- `diagnostic`: Captured errors or system diagnostics.
- `degraded-state`: Evidence showing a failure mode or fallback triggering.
- `trust-evidence`: Cryptographic attestations or runtime integrity claims.
