# Release Maturity Summary

This document reports the versioning status, artifact signatures, and reproducibility evidence for NemoClaw releases.

## Versioning Status

### Current Version

| Field | Value |
|-------|-------|
| Status | Alpha |
| Version scheme | Semantic Versioning (SemVer) |
| Current version | See `package.json` |

### Versioning Policy

- Major version zero (0.y.z) indicates the project is in initial development.
- Interfaces may change without notice during alpha.
- Breaking changes are communicated through release notes and upgrade procedures.
- Patch releases address bugs and security fixes.
- Minor releases add features without breaking changes.
- Major releases introduce breaking changes and follow upgrade doctrine procedures.

### Commit Versioning

All commits follow Conventional Commits format:

```text
<type>(<scope>): <description>
```

Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `ci`, `perf`, `merge`

Commit messages are enforced by commitlint via the prek `commit-msg` hook.

## Artifact Signatures

### Built Artifacts

| Artifact | Location | Verification |
|----------|----------|--------------|
| CLI distribution | `dist/` | Generated by `npm run build` |
| Plugin distribution | `nemoclaw/dist/` | Generated by `cd nemoclaw && npm run build` |
| Blueprint manifests | `nemoclaw-blueprint/` | YAML validation |
| Package checksums | Generated at release time | SHA-256 |

### Package Integrity

- `package.json` files define exact dependency versions.
- `package-lock.json` files are committed and verified at install time.
- `npm ci` produces deterministic installs from lock files.

### Git Artifacts

| Artifact | Purpose |
|----------|---------|
| Git tags | Mark release points |
| Release branches | Track stable versions |
| `main` branch | Current development |

## Reproducibility Evidence

### Build Reproducibility

The following steps produce identical output from the same source:

```console
$ git checkout <release-tag>
$ npm ci
$ npm run build
$ cd nemoclaw && npm ci && npm run build && cd ..
```

Build outputs are deterministic when:

- Node.js version matches the minimum required (v22.16+).
- Dependencies are installed from `package-lock.json`.
- No local modifications exist in the working tree.

### Test Reproducibility

Tests produce consistent results when:

- External dependencies are mocked (no real API calls in unit tests).
- Environment variables are set consistently.
- File system state is isolated per test.

### Verification Commands

Run these commands to verify a release:

| Command | Purpose |
|---------|---------|
| `git checkout <tag>` | Check out release source |
| `npm ci` | Deterministic install |
| `npm run build` | Build CLI |
| `cd nemoclaw && npm ci && npm run build && cd ..` | Build plugin |
| `make check` | Run all linters |
| `npm test` | Run all tests |
| `npx prek run --all-files` | Run all hooks |

### Coverage Thresholds

Coverage thresholds are tracked in `ci/coverage-threshold-*.json`. New code must not decrease CLI or plugin coverage below the tracked thresholds.
